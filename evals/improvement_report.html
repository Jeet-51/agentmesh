03 Changes Made
TIER 1 Metric Fixes — Core Quality Signals
1

Hallucination metric — complete redesign

Old metric scanned sentence text for source-tag keywords and consistently scored 0.42 because reports use structured JSON citations, not inline tags. Redesigned to count citations vs content paragraphs — now correctly rewards well-cited reports.
Before: keyword scan → static 0.42 After: score = min(1.0, n_citations / n_paragraphs) → 1.00
Fixed
2

Quantitative metric — extract real figures from citations

Metric defaulted to 1.0 when no yfinance data was present (inflated scores). Added _extract_yfinance_data() to pull actual numeric figures from yfinance citation claim text, including B/T/M/K suffixed values.
Before: no yfinance_data → default 1.0 (false positive) After: _extract_yfinance_data(sources) → real ground truth regex matches "$189.50", "2.93T", "391.04B"
Fixed
3

Freshness metric — add published_at to Citation model

Citations had no published_at field so freshness always returned the 0.50 neutral default. Added published_at to the shared Citation model and wired it through NewsAPI (publishedAt) and Finnhub (Unix timestamp → ISO).
shared/models.py: published_at: str | None = Field(default=None) synthesis/agent.py: "published_at": a.get("publishedAt", "").strip() datetime.fromtimestamp(ts, tz=utc).strftime(...)
Fixed
4

Finnhub data source — restructured to return dated articles

get_finnhub_data() previously returned a plain string with no metadata. Restructured to return a dict containing both the text and a list of article dicts with title, URL, and ISO published_at timestamps for freshness scoring.
Before: return text (str) After: return {"text": text, "articles": [ {"title": ..., "url": ..., "published_at": "2026-05-04T10:00:00Z"} ]}
Fixed
5

yfinance synthesis prompt — enforce actual figures in citations

The synthesis LLM was not including numeric values in yfinance citation claims, making quantitative spot-checking impossible. Added a CRITICAL section to the synthesis prompt mandating exact figures.
CRITICAL: yfinance citation claims MUST contain actual figures. Example: "Stock price: $189.50, Market cap: $2.93T, Revenue TTM: $391.04B" Do NOT write vague claims like "stock data retrieved".
Improved
TIER 2 Report Quality Metrics
6

narrative_length metric — word count + section completeness

New metric scoring report structure: 60% from word-count band score (ideal 400–900 words) and 40% from section completeness (5 expected headers: Executive Summary, Key Findings, Risk Analysis, Data Sources, Conclusion).
score = 0.6 * word_band_score + 0.4 * section_completeness Expected sections: Executive Summary, Key Findings, Risk Analysis, Data Sources, Conclusion
Added
7

Regression runner — retry logic for failed queries

Added automatic retry: if a query returns failed/timeout status or has an empty narrative, the runner waits 20s and re-submits once. Eliminated silent 0.00/0.00 score rows that corrupted aggregate averages.
if bad_status or empty_report: await asyncio.sleep(20) result = await run_single_query(client, query) # retry once
Fixed
TIER 3 Diagnostic Metrics — Tool & Calibration Signals
8

tool_activation — fraction of 5 tools used

Measures how many of the 5 available tools (yfinance, newsapi, wikipedia, edgar, finnhub) are represented in report citations. Scores 1.0 only when all 5 fire — exposes queries where orchestrator routing is weak.
activated = {c["tool_used"] for c in citations if c.get("tool_used")} score = len(activated) / 5 → avg 0.68 across reports
Added
9

citation_density — citations per 200 words

Measures how well-supported the narrative is throughout. Ideal is at least 1 citation per 200 words. Scores 1.0 when density meets or exceeds target — avg 0.91 shows reports are well-cited.
score = min(1.0, n_citations / max(3, word_count / 200)) → avg 0.91 across reports
Added
10

confidence_calibration — self-reported vs eval gap

Diagnostic-only metric (not included in overall). Measures gap between the synthesis agent's self-reported confidence score and the actual eval overall. Small gap = well-calibrated agent.
gap = abs(self_reported_confidence - eval_overall) score = max(0, 1 - gap / 0.5) → avg 0.75 — agent is reasonably well-calibrated
Added
INFRA Eval Infrastructure — Storage, Gateway, Dashboard
11

SQLite eval storage — evals/storage/eval_db.py

Built persistent SQLite database for eval results. Two tables: reports (full report JSON, sources, confidence) and eval_scores (metric rows keyed by report_id). Supports EVAL_DB_PATH env var override.
Tables: reports(report_id, query, report_json, sources_json, confidence) eval_scores(report_id, query, confidence, timestamp, metric, score, details)
Added
12

Gateway eval wiring — fire-and-forget background scoring

Added _run_eval_background() to gateway/main.py. After every completed run, the gateway fires an asyncio background task that extracts report data and runs the full eval suite in a thread executor without blocking the response.
asyncio.create_task(_run_eval_background( run_id, query, narrative, sources, yfinance_data, confidence ))
Added
13

Docker volume + env var for eval DB

Added Docker volume mount so the gateway container can read/write the eval SQLite DB on the host filesystem. Added EVAL_DB_PATH env var so scores persist across container restarts.
volumes: - ./evals:/app/evals environment: EVAL_DB_PATH: /app/evals/agentmesh_evals.db
Added
14

Regression runner — automated end-to-end test suite

Built evals/regression_runner.py to auto-submit all hard prompts, auto-approve human checkpoints, poll until terminal status, and print a pass/fail summary. Supports --start N for resume and --dry-run.
python evals/regression_runner.py # run all 10 python evals/regression_runner.py --start 7 # resume from #8 python evals/regression_runner.py --dry-run # preview only
Added
15

Dashboard CLI — coloured ANSI score table

Built evals/dashboard.py with colour-coded scores (green ≥0.75, yellow ≥0.50, red below), per-report diagnostic rows, and aggregate averages across all runs. Auto-detects Tier 3 metrics and adjusts layout.
python evals/dashboard.py # last 10 reports python evals/dashboard.py --limit 20 # last 20 reports
Added
